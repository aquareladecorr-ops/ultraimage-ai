                                                          8. LOGICA DE SINCRONIZACAO DE PAGAMENTOS
                                                          ----------------------------------------------------------------
                                                          Problema historico: MP nao estava chamando o webhook.
                                                          Solucao implementada: sync-payment robusto como fallback.

                                                          Fluxo do sync-payment (POST /api/sync-payment):
                                                            1. Busca todos os payments com status=pending do usuario
                                                              2. Para cada um: tenta buscar no MP por mp_payment_id
                                                                3. Se nao tem mp_payment_id: busca por external_reference na API MP
                                                                  4. Se encontrar aprovado: credita e atualiza o banco
                                                                    5. Retorna { credited: N } -- numero de creditos adicionados

                                                                    credits-panel.tsx detecta hasPending:
                                                                      - Se ha pagamentos pendentes: dispara syncPayments() automaticamente
                                                                        - Mostra banner amarelo com botao "Sincronizar"
                                                                          - Exibe resultado da sincronizacao
